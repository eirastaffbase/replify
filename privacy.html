Privacy Notice – Replify for Staffbase

Last updated: April 8, 2026  ·  Contact: eira.tope@staffbase.com

Replify is an internal Staffbase tool. This notice explains what data is collected, how it is stored, and how long it is kept. Processing is based on legitimate interest (support and security monitoring).

What is stored and for how long

DataHow storedKept for
Work email Plain text — accessible only to the tool owner via direct database access 365 days from last activity
Role & region Plain text 365 days from last activity
Device ID Random anonymous ID generated on install, stored locally in your browser 90 days per activity log entry
Usage events (which environment, which action) Action type + environment slug + device ID + email (if registered) 90 days, then automatically deleted
Staffbase API key Stored locally in your browser. Temporarily held server-side only during multi-step background jobs (e.g. LinkedIn article scraping) and deleted immediately after the job completes. Duration of the background job only.

What is not stored

  • Staffbase passwords or API keys (beyond temporary job use described above)
  • Staffbase customer or employee data from the environments you work in
  • Anything outside of Staffbase domains

Who can access the data

Only the tool owner (Eira Tope). The extension client cannot read stored data — all writes go through server-side functions. Data is stored on Supabase under a signed Data Processing Agreement.

Your rights

You can request access to or deletion of your data at any time by emailing eira.tope@staffbase.com. Deletion will be completed within 30 days.
